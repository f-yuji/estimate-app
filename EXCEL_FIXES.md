# Excel修正記録

対象原本: `見積書テンプレ - コピー.xlsm`（原本は変更していません）  
アプリ用修正版: `templates/estimate-template-fixed.xlsm`

## 修正内容

前後行（9行目・12行目）の配列式と比較し、各行の明細名セルB列を検索キーにする規則を確認しました。

| シート | セル | 問題 | 修正 |
|---|---|---|---|
| 内訳明細書 | K10 | 第2検索範囲だけB11参照 | すべてB10参照 |
| 内訳明細書 | M10 | B11参照 | B10参照 |
| 内訳明細書 | K11 | 第2検索範囲が#REF! | B11参照 |
| 内訳明細書 | M11 | #REF! | B11参照 |
| 内訳明細書 (2) | K10 | 2つの検索範囲がB11参照 | すべてB10参照 |
| 内訳明細書 (2) | M10 | B11参照 | B10参照 |
| 内訳明細書 (2) | K11 | 2つの検索範囲が#REF! | B11参照 |
| 内訳明細書 (2) | M11 | #REF! | B11参照 |

K列は物件概要2シートのK/L列、M列は物件概要(2)のM/N列を検索する配列式です。検索範囲・戻り範囲は前後行と同一に保ち、検索対象の明細名セルだけを当該行へ修正しました。

## 検査結果

- 13シート、数式441件を抽出。
- VBA (`xl/vbaProject.bin`) あり。修正版への保持を確認。
- 図形、画像、VML、フォームコントロールを含むため、帳票の最終再計算とPDF化はExcel COMを使用。
- `物件概要!L34` と `O34` は左右屋根勾配率の平均を各屋根面へ反映する意図を確認し、変更していません。
- 修正前の数式中エラーは上記K11/M11に存在。修正後は `#REF!`、`#VALUE!`、`#NAME?` なし。

解析データは `analysis/workbook_report.json` と `analysis/formulas.json` に保存しています。
